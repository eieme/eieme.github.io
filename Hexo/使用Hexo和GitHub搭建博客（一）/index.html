为什么使用 Hexo

  1. 快速、简洁且高效。
  2. 使用 Markdown 解析文章。
  3. 直接部署在 GitHub 上，不用去买空间，省钱。
  4. 光这三点，理由就很充分了好吗！

###安装
Hexo 需要 Node.js 和 Git 的支持，如果没有安装，请自行下载安装。
安装完以后，就可以使用 npm 完成 Hexo 的安装

如果在使用 npm 过程中，速度很慢，可以试试 更换 npm 源

开始建站

安装完之后，我们需要创建一个文件夹（如D:\Hexo）放置 Hexo 工程，然后我们使用 Git Shell 进入这个文件夹，执行命令生成建立网站所需要的所有文件。

配置 SSH keys

我们如何让本地git项目与远程的github建立联系呢?用SSH keys。
检查 SSH keys的设置，首先我们需要检查你电脑上现有的ssh key，使用 Git Shell 输入：

如果提示：No such file or directory 说明你是第一次使用git。
生成新的SSH Key：

Generating public/private rsa key pair.
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>

注意1: 此处的邮箱地址，你可以输入自己的邮箱地址;
注意2: 此处的「-C」的是大写的「C」
然后系统会要你输入密码：

Enter passphrase (empty for no passphrase):<输入加密串> //在输入的时候不会有显示任何文字符号，直接输入就好
Enter same passphrase again:<再次输入加密串>

在回车中会提示你输入一个密码，这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容。
最后看到这样的界面，就成功设置ssh key了：

添加 SSH Key 到 GitHub

在本机设置SSH Key之后，需要添加到 GitHub 上，以完成 SSH 链接的设置。

  1. 打开本地 C:\Users\Administrator.ssh\id_rsa.pub 文件。此文件里面内容为刚才生成人密钥。如果看不到这个文件，你需要设置显示隐藏文件。准确的复制这个文件的内容，才能保证设置的成功。
  2. 登陆 GitHub 系统。点击右上角的 Settings—>SSH and GPG keys —> New SSH key
  3. 把你本地生成的密钥复制到里面(key文本框中)， 点击 add key 就ok了
